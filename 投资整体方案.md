## 一、三大路线总览

| 路线                | 本质                         | 难度                 | 适合你的原因                                     |
| :------------------ | :--------------------------- | :------------------- | :----------------------------------------------- |
| **A. 盘中实时监控** | 网页仪表盘 + 自动刷新        | ⭐⭐ 纯前端部署简单    | 打开浏览器就能看，不占时间，最轻量               |
| **B. 本地数据分析** | Python + 数据接口 + 本地展示 | ⭐⭐⭐ 需要编码和数据库 | 和你的 `rent_house` 项目结构高度相似，技能可复用 |
| **C. 云端全自动**   | 定时脚本 + 数据库 + 钉钉通知 | ⭐⭐⭐⭐ 需要服务器部署  | 和你的 `find_job` 项目流程相通，但维护成本最高   |

对于 500 元的试水资金，**路线 A + B 的轻量组合**最合适：初期用 A 快速上手，投入时间后如果想深度分析，再往 B 扩展。

------

## 二、路线A：纯前端实时监控——最简单，零依赖

如果你只是想打开网页看净值，不想碰代码，这条路线最合适。

### 🔷 real-time-fund ⭐核心推荐

GitHub: `hzm0321/real-time-fund`

一个基于 Next.js 的纯前端基金估值与重仓股实时追踪工具，核心特性：

- 输入基金代码，实时获取单位净值和实时估算涨跌幅
- 自动展示前 10 大重仓股票及其盘中涨跌
- **纯前端运行**，通过 JSONP 直连东方财富等公开接口
- 支持 GitHub Pages 免费托管，真正的零成本部署
- 自选功能、可调刷新间隔（5秒-300秒）、移动端适配

### 🔷 manmankan

GitHub: `manmankan`

一个纯命令行的 A 股自选股位置感工具，把“先看历史价位再决策”的思路迁移到股市，用命令行一屏显示自选股在不同时间窗口内的位置百分位。适合喜欢终端操作、想要更纯粹看盘方式的投资者。

------

## 三、路线B：本地数据分析——与你的 `rent_house` 结构高度相似

如果你想在 Python 里分析基金数据，找历史规律，这条路线最合适。

### 🔷 FVTracker

GitHub: `htlaoyang/FVTracker`

一个基于 Python 的基金估值跟踪工具，接口数据来源于公开 API，和你的 `rent_house` 结构非常像：

- 实时净值获取、指数实时净值获取
- 历史估值查询和左侧加仓策略建议
- 基金管理模块 + 数据库（你的 `rent_house` 也是数据库 + 爬虫结构）
- 邮件提醒
- Tkinter GUI，跑在本地

### 🔷 xalpha / fund-investment-calculator

PyPI: `xalpha`、`fund-investment-calculator`

专为中文基金投资定制的 Python 库，核心支持定投策略模拟、历史回测、收益率归因与可视化。如果你有 Python 基础，直接 `pip install` 就能开始分析。

### 🔷 Wealthfolio

GitHub: `afadil/wealthfolio`

**隐私优先**的桌面投资组合追踪器，所有数据本地存储，支持 CSV 导入。UI 界面优雅，功能上比命令行类工具更直观易用。和你的 `rent_house` 都需要写入本地数据库，数据管理逻辑相通。

## 四、路线C：云端自动化通知——与你的 `find_job` 结构相通

如果你想在服务器上跑定时任务，从股票/基金数据中筛选信号自动推送到钉钉，这条路线适合你。

### 🔷 daily_stock_analysis

GitHub: `ZhuLinsen/daily_stock_analysis`

一个 LLM 驱动的 A/H/美股智能分析器，多数据源行情 + 实时新闻 + LLM 决策仪表盘 + 多渠道推送，支持将报告推送到钉钉/微信/飞书等渠道，和你的 `find_job` 推送逻辑相通。

### 🔷 VeighNa (vn.py) - 进阶选择

GitHub 3 万+ Star，一套基于 Python 的开源量化交易系统开发框架，功能完整。

> ⚠️ **提醒**：量化交易框架比较复杂，对个人投资者可能过于庞大，建议先从小工具开始，积累经验后再考虑。

### 🔷 Ghostfolio

GitHub: `ghostfolio/ghostfolio`

一个开源、自托管的财富管理平台，支持追踪股票、ETF、加密货币，可 Docker 自部署，功能完整且社区活跃。

## 五、数据分析的基石：免费开源金融数据 API

这几个项目共享一套核心的金融数据接口库，也是你实现上述所有功能的基础。

| 工具         | 特点                                                         | 数据范围                                                     | 适合场景                                     |
| :----------- | :----------------------------------------------------------- | :----------------------------------------------------------- | :------------------------------------------- |
| **AKShare**  | 完全免费开源，800+ API 接口，分钟级更新，GitHub 12,200+ Star | 股票/基金/期货/宏观经济，支持 A 股实时行情、历史 K 线、财务指标、基金净值 | 作为所有项目的底层数据源，性能稳定、接口丰富 |
| **yfinance** | 两行代码拉取全球股票历史行情、财务报表、分红记录             | 美股为主，全球股票                                           | 对美股 NASDAQ100 基金的分析需要它            |
| **xfinlink** | 免费美股金融数据 API，提供价格、基本面、实体解析             | 美股（全市场）                                               | 美股数据补充                                 |
| **PyWencai** | Python 金融数据接口库，面向对象设计，易上手                  | 股票/期货/期权                                               | 新手入门选择                                 |

这些库大部分都提供标准化的 **pandas DataFrame** 输出，可以直接对接你的数据分析脚本。

## 七、实用建议：从哪一个开始？

1. **最轻量的起点**（5 分钟上手）：把 `real-time-fund` 部署到 GitHub Pages，每天花 30 秒打开网页，熟悉基金估值变化的节奏。
2. **可视化加码**（30 分钟）：在本地跑起 `Wealthfolio`，导入你的 500 元持仓，跟踪收益变化。
3. **数据分析深化**：用 `AKShare` + `pandas` 写脚本分析基金历史净值、最大回撤、同类排名，技能树和你做 `rent_house` 时完全一致。
4. **自动化通知**：从 `daily_stock_analysis` 中抽取出推送模块，对设定基金的价格进行监控，符合阈值时自动推送钉钉。

从 `real-time-fund` 入手最轻量，它的 Next.js 纯前端设计很适合快速部署。等你熟悉数据逻辑后，可以基于 `AKShare` 实现更多自定义分析——这条技术路径和你搭建 `rent_house` 时的数据库+爬虫逻辑一脉相承，投入产出比最高。
