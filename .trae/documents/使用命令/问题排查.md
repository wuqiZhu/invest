# 基金投资分析工具 - 问题排查指南

## 常见问题及解决方案

---

## 一、数据获取问题

### 问题1：数据不足，无法分析

**现象**：
```
KeyError: 'modules'
数据不足30条，无法获取信号
```

**原因**：
- 基金成立时间短
- 指定的分析天数太短
- 数据源返回数据不完整

**解决方案**：
```bash
# 增加分析天数
python scripts/main_cli.py --action signal --fund 110011 --days 365

# 使用更长的天数
python scripts/daily.py --morning
```

---

### 问题2：获取基金数据失败

**现象**：
```
获取基金数据失败
网络请求失败
```

**原因**：
- 网络连接问题
- 数据源接口变化
- 基金代码错误

**解决方案**：
```bash
# 检查网络连接
ping www.baidu.com

# 检查基金代码是否正确
python scripts/main_cli.py --action list

# 尝试使用其他基金代码
python scripts/main_cli.py --action signal --fund 110030
```

---

### 问题3：东方财富API分页问题

**现象**：
```
只返回最近20条记录
数据不完整
```

**原因**：
- 东方财富API的per参数限制
- 需要分页获取数据

**解决方案**：
系统已自动处理分页，如果仍有问题：
```bash
# 使用更长的天数
python scripts/main_cli.py --action signal --fund 110011 --days 730
```

---

## 二、配置问题

### 问题4：配置文件加载失败

**现象**：
```
配置文件加载失败
FileNotFoundError: config.yaml
```

**原因**：
- 配置文件不存在
- 配置文件路径错误
- YAML格式错误

**解决方案**：
```bash
# 检查配置文件是否存在
ls scripts/*.yaml

# 使用向导模式重新配置
python scripts/main_cli.py --wizard

# 检查YAML格式
python -c "import yaml; yaml.safe_load(open('scripts/user_config.yaml'))"
```

---

### 问题5：基金列表为空

**现象**：
```
没有启用的基金，请先配置
```

**原因**：
- user_config.yaml中没有配置基金
- 基金配置被禁用

**解决方案**：
```bash
# 编辑user_config.yaml
notepad scripts/user_config.yaml

# 添加基金配置
funds:
  - code: "110011"
    name: "易方达中小盘混合"
    monthly_invest: 200
    enabled: true
```

## 四、依赖问题

### 问题8：Python版本不兼容

**现象**：
```
SyntaxError: invalid syntax
ImportError: cannot import name 'xxx'
```

**原因**：
- Python版本过低（需要3.7+）
- 使用了不兼容的语法

**解决方案**：
```bash
# 检查Python版本
python --version

# 使用Python 3.7+
python3 --version
```

---

### 问题9：缺少依赖包

**现象**：
```
ModuleNotFoundError: No module named 'xxx'
```

**原因**：
- 未安装依赖包
- 依赖包版本不兼容

**解决方案**：
```bash
# 安装依赖
pip install -r requirements.txt

# 或者只安装核心依赖
pip install pandas numpy requests pyyaml
```

---

### 问题10：AKShare/empyrical安装失败

**现象**：
```
ERROR: Could not find a version that satisfies the requirement akshare
```

**原因**：
- Python版本不兼容（需要3.9+）
- 网络问题

**解决方案**：
```bash
# 这些是可选依赖，不影响核心功能
# 可以跳过安装，使用内置的数据源

# 如果需要安装，升级Python到3.9+
python3.9 -m pip install akshare
```

**现象**：
```
钉钉通知发送失败

```

**原因**：
- webhook配置错误

  method: "dingtalk"

## 六、数据库问题

### 问题12：数据库锁定

**现象**：
```
database is locked
sqlite3.OperationalError
```

**原因**：
- 多个进程同时访问数据库
- 数据库文件被占用

**解决方案**：
```bash
# 关闭其他访问数据库的程序
# 等待几秒后重试

# 如果问题持续，删除数据库文件（会丢失历史数据）
rm scripts/invest.db
```

---

## 七、性能问题

### 问题13：命令执行缓慢

**现象**：
```
等待很长时间才返回结果
```

**原因**：
- 网络请求慢
- 数据量大
- 数据源响应慢

**解决方案**：
```bash
# 使用安静模式减少输出
python scripts/daily.py --morning --quiet

# 减少分析天数
python scripts/main_cli.py --action signal --fund 110011 --days 180

# 使用缓存（系统已自动启用requests_cache）
```

---

## 八、输出问题

### 问题14：Unicode编码错误

**现象**：
```
UnicodeEncodeError: 'gbk' codec can't encode character
```

**原因**：
- Windows控制台不支持特殊字符
- 编码问题

**解决方案**：
系统已自动处理，使用ASCII字符替代：
- ⚠️ 替换为 [!]
- ✅ 替换为 [OK]
- ❌ 替换为 [X]

---

### 问题15：输出格式混乱

**现象**：
```
输出内容对不齐
表格显示异常
```

**原因**：
- 终端宽度不够
- 字体问题

**解决方案**：
```bash
# 使用详细模式
python scripts/daily.py --morning --verbose

# 调整终端窗口大小
# 使用等宽字体
```
