## 第五阶段：整合devils-advocate到signal

### 目标
将魔鬼代言人作为signal命令的可选开关，减少独立命令。

### 修改文件
- `scripts/main_cli.py` - 添加--devil参数，修改signal方法

### 具体步骤

#### 5.1 添加--devil参数到argparse
在 `main_cli.py` 的 `main()` 函数中，添加新的命令行参数：

```python
parser.add_argument('--devil', action='store_true', 
                   help='显示反面观点（魔鬼代言人模式）')
```

位置：在现有参数列表末尾（约1009行后）

#### 5.2 修改signal方法签名
将 `signal` 方法的签名从：
```python
def signal(self, fund_code, days=365):
```
改为：
```python
def signal(self, fund_code, days=365, devil=False):
```

#### 5.3 在signal方法末尾添加魔鬼代言人逻辑
在 `signal` 方法的综合决策部分之后（约630行），添加：

```python
        # 魔鬼代言人模式
        if devil:
            print(f"\n{'='*50}")
            print(f"【魔鬼代言人】")
            print(f"{'='*50}")
            
            # 获取分析结果
            analysis = self.analyzer.analyze_fund(fund_data, fund_code)
            benchmark_code = self.config.get('analysis.benchmark_code', '110030')
            benchmark_data = self.fetcher.get_fund_nav(benchmark_code, start_date, end_date)
            
            warnings = self.analyzer.devils_advocate(fund_data, fund_code, analysis, benchmark_data)
            mistakes = self.analyzer.check_common_mistakes(fund_data, fund_code, analysis)
            
            if warnings:
                print("\n反面观点：")
                for w in warnings:
                    print(f"  - {w}")
            else:
                print("\n该基金目前没有明显的风险信号。")
            
            if mistakes:
                print("\n常见错误提醒：")
                for m in mistakes:
                    print(f"  - {m['warning']}")
                    print(f"    建议：{m['advice']}")
            
            print(f"{'='*50}")
```

#### 5.4 修改main函数中的signal调用
将第1100行的调用从：
```python
cli.signal(args.fund[0], args.days or 365)
```
改为：
```python
cli.signal(args.fund[0], args.days or 365, devil=args.devil)
```

#### 5.5 更新帮助文档示例
在epilog部分的示例中，添加：
```python
  # 模块化交易信号（含反面观点）
  python main_cli.py --action signal --fund 110011 --devil
```

### 预期收益
- 减少独立的devils-advocate命令
- signal命令更强大，可选择性显示反面观点
- 用户体验更好：看完信号后直接看反面观点

### 实现代码

#### 添加deep子命令
```python
# 创建子命令解析器
subparsers = parser.add_subparsers(dest='subcommand', help='子命令')

# deep子命令
deep_parser = subparsers.add_parser('deep', help='深度分析命令')
deep_parser.add_argument('deep_action', 
                        choices=['analyze', 'compare', 'backtest', 'simulate', 'chart', 'report'],
                        help='深度分析操作')
deep_parser.add_argument('--fund', '-f', nargs='+', help='基金代码')
deep_parser.add_argument('--days', '-d', type=int, help='分析天数')
deep_parser.add_argument('--benchmark', '-b', default='110030', help='基准基金代码')
deep_parser.add_argument('--amount', type=int, help='定投金额')
deep_parser.add_argument('--short-ma', type=int, help='短期均线周期')
deep_parser.add_argument('--long-ma', type=int, help='长期均线周期')
```

#### 精简后的主命令
```python
parser.add_argument('--action', '-a',
                   choices=['signal', 'what-if', 'note', 'config', 'wizard', 'list', 'timeline'],
                   help='执行的操作（核心命令）')
```
