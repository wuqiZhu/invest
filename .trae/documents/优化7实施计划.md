# 优化7实施计划

## 一、文件分析总结

### 1.1 核心观点

优化7.md是一份**战略性精简文档**，核心思想是：

> "工具的价值，不在于它有多少功能，而在于它是否让你更轻松地做出更好的决策。"

与之前的优化文档不同，这次是**"精简、聚焦、极致体验"**。

### 1.2 项目整体评价

#### 优点
1. **功能全覆盖**：从数据获取到信号生成，从心理保护到投研知识库
2. **工程化扎实**：requirements.txt、Docker、可选依赖、配置分离
3. **日常脚本设计**：daily.py真正回归"日常使用"场景
4. **假设分析**：what-if比复杂回测更贴近普通投资者
5. **个人笔记系统**：note命令让知识库活起来

#### 隐忧
| 问题 | 具体表现 | 潜在后果 |
|------|----------|----------|
| **功能过剩** | 20+个--action选项 | 认知负担大，真正需要的功能被淹没 |
| **配置复杂** | config.yaml上百行 | 新手配置门槛高 |
| **依赖膨胀** | 可选依赖版本要求不统一 | 环境维护成本高 |

### 1.3 核心建议

#### 保留的核心模块（每天使用）
```
daily.py (晨报、定投、复盘)
signal 命令（已优化为仓位建议）
what-if 命令（偶尔用，但很有启发）
note 命令（个人笔记，积累思考）
```

#### 降级为"扩展命令"（不显示在主帮助中）
```
analyze, compare, backtest, simulate → 合并到 "deep" 子命令
devils-advocate, buffet-bet, random-review → 独立脚本
books, quotes, check-risk → 独立脚本
```

#### 删除或合并的功能
- `--action monitor`：已被daily.py --morning覆盖
- `--action journal`独立命令：应与定投命令合并
- `--action random-review`：整合到daily.py --review中
- `--action devils-advocate`：作为signal的可选开关（--devil）

**目标**：主命令列表不超过6个

### 1.4 下一步迭代方向

| 方向 | 说明 | 优先级 |
|------|------|--------|
| 配置分级 | default_config.yaml + user_config.yaml | 高 |
| 输出优化 | 简洁格式，--verbose显示详情 | 高 |
| 安静模式 | --quiet只在重要信号时通知 | 中 |

## 三、实施计划

基于优化7.md的建议，制定以下实施计划：

### 第一阶段：精简主命令（高优先级）

**目标**：主命令列表不超过6个

**修改文件**：
- `main_cli.py` - 精简命令列表
- `daily.py` - 整合monitor、journal、review

**具体步骤**：

1. 保留的主命令（6个）
   ```
   daily    - 日常操作（晨报、定投、复盘）
   signal   - 交易信号
   what-if  - 假设分析
   note     - 投资笔记
   config   - 配置管理
   wizard   - 向导模式
   ```

2. 合并的命令
   - `monitor` → `daily.py --morning`
   - `journal` → `daily.py --invest`
   - `review` → `daily.py --review`
   - `daily-report` → `daily.py --morning`
   - `random-review` → `daily.py --review`（随机触发）

3. 降级为扩展命令
   - `analyze, compare, backtest, simulate` → `deep` 子命令
   - `devils-advocate, buffet-bet` → `signal --devil`
   - `books, quotes, check-risk` → 独立脚本 `knowledge.py`

**预期收益**：
- 主命令从23个减少到6个
- 认知负担大幅降低

---

### 第二阶段：优化daily.py输出（高优先级）

**目标**：简洁格式，一目了然

**修改文件**：
- `daily.py` - 优化输出格式

**具体步骤**：

1. 晨报输出格式
   ```
   2026-05-25 晨报
   110011 易方达中小盘：建议维持定投 (趋势↑，波动适中)
   161725 招商中证白酒：建议减半定投 (趋势↓)
   今日无操作建议。
   ```

2. 定投输出格式
   ```
   2026-05-25 定投记录
   110011 易方达中小盘：买入200元 净值1.5234
   161725 招商中证白酒：买入300元 净值0.8765
   定投完成，共投入500元。
   ```

3. 复盘输出格式
   ```
   2026-05-25 月底复盘
   本月定投：1000元
   当前市值：12500元
   总收益率：12.5%
   
   反思提示：
   1. 本月最大的市场新闻是什么？
   2. 你的信号系统给出了什么建议？
   ```

**预期收益**：
- 信息一目了然
- 减少视觉噪音

---

### 第三阶段：配置分级（中优先级）

**目标**：升级项目时不会丢失个性化设置

**修改文件**：
- `config_manager.py` - 支持分级配置
- `default_config.yaml` - 系统默认配置
- `user_config.yaml` - 用户自定义配置

**具体步骤**：

1. 创建default_config.yaml（系统内置，不修改）
   ```yaml
   # 系统默认配置
   alert:
     daily_drop_threshold: -3
     weekly_drop_threshold: -8
   
   analysis:
     default_days: 365
     risk_free_rate: 0.03
   ```

2. 创建user_config.yaml（用户自定义）
   ```yaml
   # 用户自定义配置
   funds:
     - code: "110011"
       name: "易方达中小盘混合"
       monthly_invest: 200
   
   notification:
     method: "dingtalk"
   ```

3. 修改config_manager.py支持分级配置
   ```python
   def load_config(self):
       # 1. 加载默认配置
       default_config = self._load_yaml('default_config.yaml')
       # 2. 加载用户配置
       user_config = self._load_yaml('user_config.yaml')
       # 3. 合并配置
       self.config = self._merge_config(default_config, user_config)
   ```

**预期收益**：
- 升级项目时不会丢失个性化设置
- 配置文件更简洁

---

### 第四阶段：安静模式（中优先级）

**目标**：只在重要信号时通知

**修改文件**：
- `daily.py` - 添加--quiet参数
- `main_cli.py` - 添加--quiet参数

**具体步骤**：

1. 添加--quiet参数
   ```bash
   python daily.py --morning --quiet  # 只在有重要信号时输出
   ```

2. 实现逻辑
   ```python
   def morning(cli, quiet=False):
       if quiet:
           # 只在触发预警或仓位建议变化时输出
           alerts = get_important_alerts()
           if alerts:
               send_notifications(alerts)
           else:
               print("今日无重要信号")
       else:
           # 正常输出
           print_full_report()
   ```

**预期收益**：
- 减少日常干扰
- 只关注重要信号

---
