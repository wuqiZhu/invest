# 优化6实施计划

## 一、文件分析总结

### 1.1 核心观点

优化6.md是一份**战略性转型文档**，核心思想是：

> "你不需要一个什么都能做的万能工具箱，你需要一个能陪你从500元走到5万元的贴身搭档。"

与之前的优化文档不同，这次不是"加功能"，而是**"减功能、提体验"**。

### 1.2 项目整体评价

| 维度 | 评分 | 评价 |
|------|------|------|
| **功能完整性** | ⭐⭐⭐⭐⭐ | 覆盖全面，几乎没有短板 |
| **代码质量** | ⭐⭐⭐⭐ | 模块划分清晰，但缺少单元测试 |
| **投资实用性** | ⭐⭐⭐⭐ | 方向正确，但实盘有效性未验证 |
| **个性化适配** | ⭐⭐⭐ | 功能堆叠较多，缺少裁剪机制 |
| **维护成本** | ⭐⭐⭐ | 依赖项增多，可能导致环境混乱 |

### 1.3 三个核心矛盾

#### 矛盾1：全量功能 vs. 日常高频使用

**现状**：
- 16个 `--action` 选项
- 20+ 个命令行参数
- 6 个可选依赖

**现实**：每天真正会用的命令不超过3条：
- 早上看信号：`signal --fund 110011`
- 定投日执行：`journal --trade-action buy`
- 月底复盘：`review`

**建议**：创建 `daily.py` 脚本，一键完成日常操作

#### 矛盾2：开源依赖的"诱人" vs. "维护成本"

**现状**：
- AKShare需要Python 3.9+
- 接口可能不定期变化
- 引入后增加不确定性

**建议**：AKShare放在 `extras_require`，不作为默认安装，用 `try/except ImportError` 优雅降级

#### 矛盾3：知识库模块的定位

**现状**：
- 预置12本书籍、13条金句
- 与核心投资决策是弱关联

**建议**：变成**个人笔记系统**，让用户自己添加书摘、反思

### 1.4 下一个迭代方向

| 方向 | 说明 | 优先级 |
|------|------|--------|
| 简化配置 | 分级配置：default_config.yaml + user_config.yaml | 高 |
| 操作建议 | 从"BUY/SELL/HOLD"改为建议仓位比例或定投倍数 | 高 |
| 整合日志 | 统一timeline视图，把市场事件、操作、反思放在同一条时间线 | 中 |
| 假设分析 | "如果我当年每月定投500到这支基金，现在有多少钱" | 中 |

### 第二阶段：简化配置（高优先级）

**目标**：分级配置，升级项目时不会丢失个性化设置

**修改文件**：
- `config.yaml` → `default_config.yaml` + `user_config.yaml`
- `config_manager.py` - 支持分级配置

**具体步骤**：

1. 创建default_config.yaml（系统内置，不修改）
   ```yaml
   # 系统默认配置，不要修改此文件
   # 如需自定义，请修改 user_config.yaml
   
   alert:
     daily_drop_threshold: -3
     weekly_drop_threshold: -8
   
   analysis:
     default_days: 365
     risk_free_rate: 0.03
   ```

2. 创建user_config.yaml（用户自定义，只保存修改过的）
   ```yaml
   # 用户自定义配置
   # 只需要填写你想修改的配置项
   
   funds:
     - code: "110011"
       name: "易方达中小盘混合"
       monthly_invest: 200
   
   notification:
     method: "dingtalk"
     dingtalk:
       webhook_url: "your_webhook_url"
   ```

3. 修改config_manager.py支持分级配置
   ```python
   def load_config(self):
       # 1. 加载默认配置
       default_config = self._load_yaml('default_config.yaml')
       # 2. 加载用户配置
       user_config = self._load_yaml('user_config.yaml')
       # 3. 合并配置（用户配置覆盖默认配置）
       self.config = self._merge_config(default_config, user_config)
   ```

**预期收益**：
- 升级项目时不会丢失个性化设置
- 配置文件更简洁
