## 一、模块化交易系统核心思想（图片要点提取）

图片把交易系统拆成**五个独立模块**，每个模块只回答一个决策问题：

| 模块         | 决策问题     | 常用工具（图片内）                         | 核心逻辑                               |
| :----------- | :----------- | :----------------------------------------- | :------------------------------------- |
| **趋势模块** | 做不做？     | EMA20、EMA60、MA120、周线/日线趋势         | 只在顺势方向找机会，逆势减少出手       |
| **买点模块** | 什么时候做？ | SAR、MACD金叉、K线回踩、突破回踩、均线支撑 | 在趋势方向内，等具体信号出现才进场     |
| **波动模块** | 值不值得做？ | ATR、布林带宽度、成交量放大、真实波动区间  | 波动太小不做（没空间），波动太大要小心 |
| **风控模块** | 错了亏多少？ | ATR止损、前低止损、固定亏损比例（1-2%）    | 进场前先算好止损位，控制单笔亏损       |
| **止盈模块** | 对了赚多少？ | 1.5ATR止盈、分批止盈、移动止损、压力位减仓 | 保护利润，让利润奔跑的同时不让它回吐   |

> 核心原则：**模块之间不耦合**——趋势判断错了，不影响你严格执行止损；买点错了，风控会兜底。

### 2. 新增一个模块 `trading_system.py`

把五个模块的判断逻辑独立出来，让 `main_cli.py` 调用它生成**交易信号**，而不是单纯的分析报告。

python

```
# trading_system.py
class TradingSystem:
    def __init__(self, config, fetcher, analyzer):
        self.config = config
        self.fetcher = fetcher
        self.analyzer = analyzer

    def get_signal(self, fund_code):
        """返回: 'BUY', 'SELL', 'HOLD'，并给出理由"""
        nav_data = self.fetcher.get_fund_nav(fund_code)
        if nav_data.empty:
            return 'HOLD', "无数据"
        
        # ----- 1. 趋势模块 -----
        ema20 = nav_data['单位净值'].ewm(span=20).mean().iloc[-1]
        ema60 = nav_data['单位净值'].ewm(span=60).mean().iloc[-1]
        latest_nav = nav_data['单位净值'].iloc[-1]
        trend_up = (latest_nav > ema20) and (ema20 > ema60)
        if not trend_up:
            return 'HOLD', f"趋势向下，最新净值{latest_nav:.3f} < EMA20{ema20:.3f}"
        
        # ----- 2. 波动模块 -----
        atr = self.analyzer.calculate_atr(nav_data['单位净值'], period=14)
        atr_ratio = atr / latest_nav
        if atr_ratio < 0.005:   # ATR小于0.5%价格，波动太小
            return 'HOLD', f"波动太低 ATR比率{atr_ratio:.3%}"
        
        # ----- 3. 买点模块 -----
        sar = self.analyzer.calculate_sar(nav_data['单位净值'])  # 需要实现SAR
        macd = self.analyzer.calculate_macd(nav_data['单位净值'])
        # 简化：SAR翻多且MACD金叉
        if sar.get('signal') == 'bullish' and macd.get('golden_cross'):
            # ----- 4. 风控模块 -----
            stop_loss = latest_nav - atr * self.config.get('risk.atr_stop_multiplier')
            # ----- 5. 止盈模块（先不在这里判断）-----
            return 'BUY', f"趋势向上，ATR比率{atr_ratio:.3%}，SAR翻多，止损位{stop_loss:.3f}"
        
        return 'HOLD', "趋势向上但买点未出现"
```

## 三、针对“基金”的特殊改造（图片里是股票/期货思路，需调整）

图片里的框架是为**股票/期货日内或波段交易**设计的，用在**场外基金**上有三个关键差异：

| 差异点       | 股票/期货   | 场外基金                      | 调整方案                                                     |
| :----------- | :---------- | :---------------------------- | :----------------------------------------------------------- |
| **交易频率** | 可随时买卖  | 每天只有一个净值，T+1确认     | 信号频率降到日线，取消小时线判断                             |
| **止损方式** | 可设条件单  | 无法盘中止损，只能次日赎回    | 用“持有期约束”代替止损；例如：买入后必须持有至少7天，期间不做短期止损 |
| **波动模块** | ATR决定仓位 | 基金本身已经分散，ATR意义减弱 | 改用“基金波动率 vs 基准波动率”                               |

### 专门为基金设计的五模块建议

- **趋势模块**：周线MA20 + 日线EMA20（不变）
- **买点模块**：取消SAR（基金净值不连续，SAR假信号多），改用**均线偏离+定投日**：例如净值低于EMA20超过5%，且是定投日，则加倍扣款。
- **波动模块**：用**年化波动率**替代ATR，波动率>25%属于高波动基金，减少单笔投入。
- **风控模块**：不是“止损”，而是**分散持有** + **回撤控制**：当基金从高点回撤超过15%时，停止定投，观察一个月。
- **止盈模块**：采用**目标收益率止盈**（例如收益超过20%赎回一半） + **移动回撤止盈**（从最高点回撤8%全赎回）。

你的工具里已经有最大回撤、波动率、基准对比，直接复用即可。

## 四、心理层面：模块化如何帮你克服“情绪交易”

图片里最值钱的一句话：**“交易系统越清晰，执行才越稳定”**。

你之前写的魔鬼代言人、冷静期、随机复盘，都是从外部约束你的行为。而模块化系统是从**内部结构**上让你不再纠结：

- 当你想抄底时，趋势模块告诉你“现在在均线下方，不做”，你就不会冲动。
- 当你想死扛亏损时，风控模块告诉你“已经跌破止损位，必须出场”，你就不会犹豫。
- 当你想贪婪多赚时，止盈模块告诉你“到了1.5ATR目标，减仓一半”，你就不会贪婪。

**你可以把你的脚本扩展成“交易教练”**：每次你执行 `--action signal` 后，系统强制要求你在 `journal` 里填写**哪个模块**是你的决策依据。一个月后复盘，你就能清楚地看到：你是输在趋势判断错误，还是买点太急，还是没执行止损。
